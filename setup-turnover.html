Dashboard de Turnover — Como Rodar Localmente

Guia manual para executar API, banco local e interface web.

Opção recomendada — Docker

git clone https://github.com/lucaspwalter/turnover-dashboard.git cd turnover-dashboard docker compose up

Acesse http://localhost:3000. API e banco SQLite são iniciados automaticamente.

Instalação manual rápida

Terminal 1 — API

git clone https://github.com/lucaspwalter/turnover-dashboard.git cd turnover-dashboard/backend python3 -m venv .venv source .venv/bin/activate pip install -r requirements.txt python3 main.py

Terminal 2 — Frontend

cd turnover-dashboard/frontend cp .env.example .env npm install npm start

Windows: troque python3 por py, ativação por .\.venv\Scripts\Activate.ps1 e cp por Copy-Item.

Sobre o projeto

Dashboard para identificar funcionários com maior risco de saída por meio de um score baseado em seis fatores auditáveis.

Pré-requisitos

  • Docker, somente para opção recomendada
  • Git
  • Python 3.10 ou superior
  • Node.js 18 ou superior

PostgreSQL é opcional. Sem DATABASE_URL, o projeto usa SQLite automaticamente.

Acessos

Dashboard: http://localhost:3000 Documentação da API: http://localhost:8000/docs Saúde da API: http://localhost:8000/health

Na primeira execução, a API cria o banco SQLite e popula 50 funcionários fictícios automaticamente.
