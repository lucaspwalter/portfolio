Support System — Como Rodar Localmente

Guia manual para executar o back-end, front-end e banco localmente.

Sobre o projeto

Sistema de suporte ao vivo com chat em tempo real via WebSocket STOMP. Clientes abrem sessões de atendimento, entram em fila de espera e conversam diretamente com atendentes. O backend é construído em Java com Spring Boot e o frontend em Next.js.

Pré-requisitos

  • Docker instalado e rodando
  • Java 21 ou superior
  • Maven 3.9 ou superior
  • Node.js 18 ou superior
  • Git

Linux / macOS — Passo a passo

1. Clone o repositório

git clone https://github.com/lucaspwalter/support-system.git cd support-system

2. Suba o banco de dados e o backend via Docker Compose

docker compose up -d

3. Aguarde o backend iniciar (30-60 segundos)

Verifique:

curl http://localhost:8080/agents

4. Em outro terminal, instale e inicie o frontend

cd frontend npm install npm run dev

5. Acesse:

Cliente: http://localhost:3000/client Atendente: http://localhost:3000/agent Histórico: http://localhost:3000/history

Como testar

Abra duas abas no navegador. Na primeira acesse /client e inicie um atendimento informando nome e assunto. Na segunda acesse /agent, onde a sessão aparecerá na fila. Clique em Aceitar para iniciar o chat em tempo real. As mensagens aparecem instantaneamente nos dois lados. Ao encerrar, a conversa fica disponível em /history.
