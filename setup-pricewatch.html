Como Rodar Localmente no Linux/Ubuntu e Windows

Cada etapa abaixo mostra a versão para Linux/Ubuntu e a versão para Windows.

Opção recomendada — Docker

git clone https://github.com/lucaspwalter/pricewatch.git cd pricewatch docker compose up --build

Acesse http://localhost:3000. PostgreSQL, backend e frontend são iniciados automaticamente.

1. Baixar o projeto do GitHub

Linux/Ubuntu

git clone https://github.com/lucaspwalter/pricewatch.git cd pricewatch

Windows

git clone https://github.com/lucaspwalter/pricewatch.git cd pricewatch

2. Instalar os programas necessários

Linux/Ubuntu

sudo apt update sudo apt install -y openjdk-17-jdk maven nodejs npm postgresql postgresql-contrib

Windows

  • Instale o JDK 17, Maven, Node.js e PostgreSQL.
  • Use os instaladores oficiais do Windows ou o gerenciador que você preferir.
  • Confirme depois que o java, mvn, node, npm e psql estão disponíveis no terminal.

3. Conferir a instalação

Linux/Ubuntu e Windows

java -version mvn -version node -v npm -v psql --version

4. Iniciar o PostgreSQL

Linux/Ubuntu

sudo systemctl start postgresql sudo systemctl enable postgresql

Windows

  • Abra o Services do Windows e inicie o serviço do PostgreSQL.
  • Se você usa o pgAdmin, confirme que a instância está ativa antes de seguir.

5. Criar o usuário e o banco de dados

Linux/Ubuntu

sudo -u postgres psql -c "CREATE USER pricewatch WITH PASSWORD 'pricewatch';" sudo -u postgres psql -c "CREATE DATABASE pricewatch OWNER pricewatch;" sudo -u postgres psql -c "GRANT ALL PRIVILEGES ON DATABASE pricewatch TO pricewatch;"

Windows

psql -U postgres -c "CREATE USER pricewatch WITH PASSWORD 'pricewatch';" psql -U postgres -c "CREATE DATABASE pricewatch OWNER pricewatch;" psql -U postgres -c "GRANT ALL PRIVILEGES ON DATABASE pricewatch TO pricewatch;"

Esses comandos criam:

  • Um usuário do banco chamado pricewatch
  • Uma senha chamada pricewatch
  • Um banco chamado pricewatch

Se aparecer uma mensagem dizendo que o usuário ou banco já existe, pode continuar se você já criou eles antes.

6. Configurar as variáveis do backend

Linux/Ubuntu

export DATABASE_URL=jdbc:postgresql://localhost:5432/pricewatch export DATABASE_USERNAME=pricewatch export DATABASE_PASSWORD=pricewatch export JWT_SECRET=pricewatch-secret-key-local-1234567890 export JWT_EXPIRATION_MS=86400000

Windows

$env:DATABASE_URL="jdbc:postgresql://localhost:5432/pricewatch" $env:DATABASE_USERNAME="pricewatch" $env:DATABASE_PASSWORD="pricewatch" $env:JWT_SECRET="pricewatch-secret-key-local-1234567890" $env:JWT_EXPIRATION_MS="86400000"

Essas variáveis estão certas se você usou exatamente os comandos do passo anterior.

Se você criou o banco com outro nome, usuário ou senha, troque estes valores.

Linux/Ubuntu

export DATABASE_URL=jdbc:postgresql://localhost:5432/NOME_DO_SEU_BANCO export DATABASE_USERNAME=SEU_USUARIO_DO_POSTGRES export DATABASE_PASSWORD=SUA_SENHA_DO_POSTGRES export JWT_SECRET=uma-chave-local-com-pelo-menos-32-caracteres export JWT_EXPIRATION_MS=86400000

Windows

$env:DATABASE_URL="jdbc:postgresql://localhost:5432/NOME_DO_SEU_BANCO" $env:DATABASE_USERNAME="SEU_USUARIO_DO_POSTGRES" $env:DATABASE_PASSWORD="SUA_SENHA_DO_POSTGRES" $env:JWT_SECRET="uma-chave-local-com-pelo-menos-32-caracteres" $env:JWT_EXPIRATION_MS="86400000"

Para ver os bancos que existem no seu PostgreSQL:

sudo -u postgres psql -c "\l"

Para ver os usuários que existem no seu PostgreSQL:

sudo -u postgres psql -c "\du"

O JWT_SECRET não vem do banco. Ele é uma chave usada pelo sistema para gerar o login. Para rodar localmente, pode usar a chave de exemplo acima.

7. Rodar o backend

Linux/Ubuntu e Windows

mvn spring-boot:run

Quando estiver funcionando, a API ficará disponível em:

http://localhost:8080

Deixe esse terminal aberto.

8. Abrir outro terminal e entrar novamente no projeto

cd pricewatch

Isso vale para Linux/Ubuntu e Windows.

9. Instalar as dependências do frontend

cd frontend npm install

Esse passo vale para Linux/Ubuntu e Windows.

10. Rodar o frontend

Linux/Ubuntu e Windows

npm run dev

Quando estiver funcionando, o site ficará disponível em:

http://localhost:3000

11. Acessar no navegador

http://localhost:3000

12. Comandos úteis

Parar o backend ou frontend:

Ctrl + C

Rodar somente os testes do backend:

cd pricewatch mvn test

Gerar o arquivo final do backend:

cd pricewatch mvn clean package

Rodar o backend com Docker

cd pricewatch docker build -t pricewatch . docker run --rm -p 8080:8080 \ -e DATABASE_URL=jdbc:postgresql://host.docker.internal:5432/pricewatch \ -e DATABASE_USERNAME=pricewatch \ -e DATABASE_PASSWORD=pricewatch \ -e JWT_SECRET=pricewatch-secret-key-local-1234567890 \ pricewatch

No PowerShell, troque cada \ de continuação por `.
